# 작업 목록: [FEATURE NAME]

**입력**: `/specs/[###-feature-name]/`의 설계 문서
**선행 조건**: plan.md(필수), spec.md(필수), research.md, data-model.md, contracts/

**테스트**: geocoding/공간쿼리/판정규칙/API 계약/데이터 정규화에 영향이 있으면 테스트는 필수
(최소: 단위 1개 + 통합 1개).

**구성 원칙**: 작업은 사용자 스토리별로 묶어 각 스토리를 독립 구현/검증 가능하게 한다.

## 형식: `[ID] [P?] [Story] 설명`

- **[P]**: 병렬 수행 가능
- **[Story]**: 해당 작업의 사용자 스토리(US1, US2, US3 등)
- 설명에는 정확한 파일 경로를 포함

## 경로 규칙

- 단일 프로젝트: `src/`, `tests/`
- 웹 앱: `backend/src/`, `frontend/src/`
- 모바일: `api/src/`, `ios/` 또는 `android/`
