# 기능 명세: 200m 대형광고물 반경 판정

**기능 브랜치**: `001-billboard-radius-check`
**작성일**: 2026-03-05
**상태**: Draft
**입력**: 사용자 설명: "주소 입력 시 200m 반경 내 대형광고물(옥상/벽면) 존재 여부를 판정하고 지도와 목록으로 제공. 기본 데이터는 2025-12-16 현황 엑셀 사용"

## Clarifications

### Session 2026-03-05

- Q: 지오코딩 결과가 여러 개일 때 판정 기준을 어떻게 고정할까요? → A: 후보 주소 목록(최대 5개)을 보여주고 사용자가 1개 선택 후 판정
- Q: 반경 판정에 포함할 광고물 상태 범위를 어떻게 정할까요? → A: 유효(운영중) 상태만 포함
- Q: 기본 데이터 갱신 방식은 어떻게 고정할까요? → A: 운영자가 수동 업로드로만 갱신(배치 자동동기화 없음)

## 사용자 시나리오와 테스트 *(필수)*

### 사용자 스토리 1 - 주소 기반 200m 판정 조회 (우선순위: P1)

행정 담당자가 주소를 입력하면 시스템은 좌표를 변환하고, 입력 지점 기준 200m 내 대형광고물 존재 여부와 목록(거리 포함)을 즉시 제공한다.

**우선순위 근거**: 허가 검토의 핵심 판단 기능이며 단독으로도 업무 가치가 즉시 발생한다.
**독립 테스트**: 주소 1건을 입력해 결과 건수, 거리, 판정(존재/없음)이 기대값과 일치하는지 확인한다.

**인수 시나리오**:

1. **Given** 유효한 주소와 반경 판정 API가 준비됨, **When** 사용자가 주소를 검색함, **Then** 시스템은 200m 내 광고물 건수와 목록(거리 포함)을 반환해야 한다.
2. **Given** 반경 내 광고물이 없는 주소, **When** 사용자가 검색함, **Then** 시스템은 건수 0과 "해당 없음" 상태를 반환해야 한다.

---

### 사용자 스토리 2 - 지도 시각화 확인 (우선순위: P2)

담당자는 지도에서 검색 위치, 200m 원, 기존 광고물 마커를 확인하고 결과 목록과 위치를 함께 검토할 수 있다.

**우선순위 근거**: 판정 결과의 신뢰성과 설명 가능성을 높여 검토 시간을 단축한다.
**독립 테스트**: 검색 후 지도에 3요소(검색 마커/200m 원/광고물 마커)가 표시되고 목록과 마커 개수가 일치하는지 확인한다.

**인수 시나리오**:

1. **Given** 반경 내 광고물이 2건인 검색 결과, **When** 결과가 렌더링됨, **Then** 지도 마커 2개와 목록 2개가 일치해야 한다.

---

### 사용자 스토리 3 - 기본 데이터 적재 및 이력 추적 (우선순위: P3)

운영자는 기본 데이터 파일을 기준으로 광고물 데이터를 적재하고, 어떤 원본 파일로부터 데이터가 생성되었는지 추적할 수 있다.

**우선순위 근거**: 운영 신뢰성과 재현성을 위해 데이터 계보 관리가 필요하다.
**독립 테스트**: 지정 원본 파일을 적재한 뒤, 데이터 행 수와 적재 이력(source file, 실행 시각, 규칙 버전)을 조회한다.

**인수 시나리오**:

1. **Given** 원본 엑셀 파일이 존재함, **When** 적재 작업을 수행함, **Then** 광고물 데이터와 적재 이력이 함께 저장되어야 한다.

### 엣지 케이스

- 주소가 모호해 지오코딩 결과가 여러 개인 경우, 시스템은 후보 주소를 최대 5개 제공하고 사용자 선택 후 판정해야 한다.
- 지오코딩 실패(존재하지 않는 주소, 외부 API 장애) 시 시스템은 오류 코드를 반환하고 재시도 가이드를 제공해야 한다.
- 동일 주소에 복수 광고물이 있는 경우, 각 광고물을 개별 항목으로 반환해야 한다.
- 거리값이 정확히 200.0m인 광고물은 포함(`<=200m`)으로 판정해야 한다.
- 데이터에 위/경도가 없거나 주소가 비정상인 행은 적재에서 제외하고 오류 로그로 남겨야 한다.

## 요구사항 *(필수)*

### 기능 요구사항

- **FR-001**: 시스템은 입력 주소를 좌표로 변환해 반경 200m 판정을 수행해야 한다.
- **FR-002**: 시스템은 PostGIS 기반 거리 계산으로 200m 이내 광고물을 조회해야 한다.
- **FR-003**: 시스템은 조회 결과를 건수, 목록(업체명/광고물종류/주소/거리) 형태로 반환해야 한다.
- **FR-004**: 시스템은 검색 위치 마커, 200m 원, 광고물 마커를 지도에 표시해야 한다.
- **FR-005**: 시스템은 기본 원본 데이터(`/mnt/c/Users/user/sign_map/data/raw/billboards_rooftop_wall_2025-12-16.xlsx`)를 적재할 수 있어야 한다.
- **FR-006**: 시스템은 적재 이력(원본 파일명, 실행 시각, 규칙 버전, 성공/실패 건수)을 저장해야 한다.
- **FR-007**: 시스템은 에러를 `VALIDATION_ERROR`, `GEOCODING_ERROR`, `INTERNAL_ERROR`로 구분해 반환해야 한다.
- **FR-008**: 시스템은 법정동/광고물종류(옥상전광/벽면전광) 기준 필터 조회를 지원해야 한다.
- **FR-009**: 시스템은 지오코딩 결과가 복수일 때 후보 주소를 최대 5개까지 제시하고, 사용자 선택 완료 후 반경 판정을 수행해야 한다.
- **FR-010**: 시스템은 반경 판정 시 광고물 상태가 `유효(운영중)`인 데이터만 조회 대상으로 포함해야 한다.
- **FR-011**: 시스템의 기본 데이터 갱신은 운영자 수동 업로드 방식으로만 수행하며, 자동 배치 동기화는 지원하지 않는다.

### 헌장 정합성 *(필수)*

- 거리 판정은 DB(PostGIS)에서 `ST_DWithin` + meter 기반 geography 계산으로 수행한다.
- 데이터 원천은 `data/raw`의 원본 파일을 기준으로 하며, 변환/적재는 재현 가능한 스크립트로 수행한다.
- API 오류 응답은 기계 판독 가능한 안정 코드와 메시지 필드를 포함한다.
- 반경 판정 API의 성능 목표는 합의된 기준 부하에서 p95 1초 이내로 정의한다.

### 핵심 엔터티 *(데이터가 있으면 필수)*

- **Billboard**: 대형광고물 엔터티. 속성은 연번, 광고물종류, 업체명, 허가일자, 규격, 표시주소, 법정동, 위도/경도, 공간좌표(geom), 상태를 포함한다.
- **RadiusCheckRequest**: 반경 판정 요청 엔터티. 속성은 입력 주소, 변환 좌표, 요청 시각, 요청자 식별자(선택), 반경값(기본 200m)을 포함한다.
- **RadiusCheckResult**: 반경 판정 결과 엔터티. 속성은 요청 ID, 판정 건수, 광고물 목록, 거리값, 판정 상태를 포함한다.
- **ImportJobLog**: 데이터 적재 이력 엔터티. 속성은 원본 파일명, 파일 해시(선택), 실행 시각, 성공/실패 건수, 오류 요약, 규칙 버전을 포함한다.

## 성공 기준 *(필수)*

### 측정 가능한 결과

- **SC-001**: 담당자는 주소 입력 후 3단계(검색-지도확인-판정확인) 이내로 결과를 확인할 수 있어야 한다.
- **SC-002**: 반경 판정 API는 기준 부하(동시 사용자 50명, 초당 100요청, 10분 지속, 데이터 10,000건)에서 p95 응답시간 1초 이내를 달성해야 한다.
- **SC-003**: 고정 검증 데이터셋(`/mnt/c/Users/user/sign_map/data/validation/radius_check_validation_2026-03-05.csv`, 총 500케이스, 정답셋 컬럼 포함) 기준 거리 판정 정확도는 99% 이상이어야 한다.
- **SC-004**: 허가 검토 1건당 평균 확인 시간을 기존 수작업 대비 50% 이상 단축해야 한다.
